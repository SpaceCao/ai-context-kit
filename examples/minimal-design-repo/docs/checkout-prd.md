# Checkout PRD (Excerpt)

## Goals

- Improve mobile checkout completion rate
- Reduce form errors during address entry

## Confirmed requirements

- Users must be able to review cart contents before payment
- Address entry must support inline validation
- Delivery selection must happen before payment
- Confirmation page must show order number and summary

## Open discussion

- Whether coupon entry should appear early or late in the flow
- Whether guest checkout should be the default entry point
